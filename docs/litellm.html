Compute Adapter

LiteLLM

Proxy

Overview

LiteLLM is a unified proxy supporting 100+ LLM providers. Add the 0G Compute Network as a provider to route requests through the adapter.

Prerequisites

  • LiteLLM installed (pip install litellm)
  • 0G Compute Adapter running on http://localhost:8000

Python SDK

import litellm

response = litellm.completion(
    model="openai/zai-org/GLM-5-FP8",
    messages=[{"role": "user", "content": "Hello from 0G!"}],
    api_base="http://localhost:8000/v1",
    api_key="your-0g-api-key",
)
print(response.choices[0].message.content)

LiteLLM Proxy Config

Add 0G models to your litellm_config.yaml:

model_list:
  - model_name: glm-5
    litellm_params:
      model: openai/zai-org/GLM-5-FP8
      api_base: http://localhost:8000/v1
      api_key: your-0g-api-key

  - model_name: deepseek-v3
    litellm_params:
      model: openai/deepseek/deepseek-chat-v3-0324
      api_base: http://localhost:8000/v1
      api_key: your-0g-api-key

  - model_name: gpt-oss-120b
    litellm_params:
      model: openai/openai/gpt-oss-120b
      api_base: http://localhost:8000/v1
      api_key: your-0g-api-key

  - model_name: qwen3-vl
    litellm_params:
      model: openai/qwen/qwen3-vl-30b-a3b-instruct
      api_base: http://localhost:8000/v1
      api_key: your-0g-api-key

Load Balancing with Fallback

Use LiteLLM's routing to combine OpenAI and 0G models with automatic fallback:

model_list:
  - model_name: smart-model
    litellm_params:
      model: gpt-4o
      api_key: sk-openai-key

  - model_name: smart-model
    litellm_params:
      model: openai/zai-org/GLM-5-FP8
      api_base: http://localhost:8000/v1
      api_key: your-0g-api-key

router_settings:
  routing_strategy: latency-based-routing
  num_retries: 2
